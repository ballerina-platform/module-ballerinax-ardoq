# Application Dependency Mapping

This example demonstrates how to model an application dependency in Ardoq. It:

1. Finds a workspace by name
2. Discovers the component and reference types defined by the workspace model
3. Creates two application components
4. Links them with a dependency reference

## Prerequisites

1. Generate an Ardoq API token as described in the [Ardoq API documentation](https://developer.ardoq.com/docs/api-and-integrations/rest-api).

2. Build and push the connector to your local Ballerina repository:

    ```bash
    cd ../../ballerina
    bal pack && bal push --repository=local
    ```

3. Create a `Config.toml` in this directory with your credentials:

    ```toml
    token = "<your-ardoq-api-token>"
    serviceUrl = "https://app.ardoq.com/api/v2"
    workspaceName = "Application Portfolio"
    ```

   > **Note:** Replace `"Application Portfolio"` with the name of an existing workspace in your Ardoq organization.

## Running the example

Execute the following commands from this directory:

```bash
bal run
```

## Expected output

```
Using workspace: Application Portfolio (6f9b2c3d4e5f6a7b8c9d0e1f)
Created component: Payment Service (7a0c3d4e5f6a7b8c9d0e1f2a)
Created component: Ledger Service (8b1d4e5f6a7b8c9d0e1f2a3b)
Created reference: 9c2e5f6a7b8c9d0e1f2a3b4c (Payment Service -> Ledger Service)
```

The IDs will differ in your environment, as they are generated by Ardoq.
